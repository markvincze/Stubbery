The same ISetup-instance, so that more conditions can be set fluently.
Remarks

If a route template gets matched, the route arguments are stored in the HttpContext, and they can be accessed through the Route property when creating responses.

If multiple routes are set up, then they'll be matched one by one until the first match found. The route arguments in Route will be populated based on the first match.
